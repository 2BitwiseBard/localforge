      Storage Paths
      

        

          
📁 Datasets

          ~/Development/training/datasets/
          
JSONL files prepared from git history, codebases, HuggingFace, or feedback

        

        

          
🧠 Training Runs & LoRA Adapters

          ~/Development/training/runs/<run-name>/
          
adapter_config.json + adapter_model.safetensors (loadable LoRA), plus GGUF exports

        

        

          
📦 GGUF Exports

          /mnt/models/<run-name>-<quant>.gguf
          
Auto-placed in models drive by sync_models. Swap in via Config tab or local call swap_model

        

        

          
🤖 HuggingFace Cache

          ~/.cache/huggingface/hub/
          
Full-precision base model weights (4–27 GB each) downloaded on first training run. Delete to free space after exporting GGUF.

        

        

          
💬 Feedback Data

          ~/Development/training/feedback.jsonl
          
DPO preference pairs from the Feedback tab. Use From Feedback dataset mode to turn into training data.

        

      

    


    
    

      How Fine-Tuning Works
      
Training produces a LoRA adapter or full GGUF you can swap in like any other model.

      

        

          
💾

          
Your data

          
git, code, HF dataset, feedback

        

        

          
→

          
Unsloth

          
downloads HF weights, trains QLoRA

        

        

          
→

          
GGUF output

          
q4_k_m, q8_0 — swap in immediately

        

      

      
Your installed GGUFs are quantized (compressed) — you can't train on them directly. Unsloth downloads full-precision HuggingFace weights (~4–14 GB) on first run, trains a LoRA adapter on top, then merges and exports a new GGUF.

    


    
    

      Feedback & DPO Data
      
Rate model responses to build preference training data for DPO fine-tuning.
